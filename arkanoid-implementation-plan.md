# Arkanoid Web (Desktop) - План реализации

## Goal
Собрать playable-версию Arkanoid на `HTML5 + CSS3 + JavaScript`, визуально и по ощущению максимально близкую к референсу `Arkanoid (Famicom, 1986)`.

## Зафиксированные требования
- Игра должна выглядеть хорошо (целостный визуальный стиль, читаемые UI-элементы).
- Игра должна быть анимированной (движение мяча/платформы, разрушение блоков, эффекты).
- Поведение должно быть максимально близко к референсу.
- Достаточно только desktop-версии (без адаптива под мобильные).
- `HTML`, `CSS`, `JS` должны быть в разных файлах.
- Допустимы вспомогательные JS-библиотеки (`jQuery`, `PixiJS`, и т.д.).
- Функционал должен быть изолирован (класс/функция/объект, без конфликтов в глобальной области).
- Бонус: этап обучения (tutorial) или дополнительные анимации.
- Обязательны звуковые эффекты.
- Нужен `high score` через `localStorage`.
- Предпочтительно fullscreen desktop-поле (без отдельной мобильной адаптации).
- По `F5` должен идти сценарий `build -> run`, запуск не через dev-server.

## Выбранный технический подход (предварительно)
- Рендер: `HTML5 Canvas` (vanilla JS, без обязательных библиотек).
- Архитектура: класс `ArkanoidGame` + внутренние сущности (`Paddle`, `Ball`, `Brick`, `Level`, `HUD`).
- Изоляция: модульный JS (`type="module"`) и отсутствие глобальных переменных.
- Файлы: `index.html`, `styles.css`, `game.js`, опционально `assets/*`.
- Сборка: `build` в папку `dist` (production-файлы).
- Запуск: отдельный production static server (`run`) и конфиг `F5` в VS Code.
